1. Executive Summary


Current State: Grain v1.0-alpha has a well-defined specification with 10 universal primitives, G-Lang syntax, state machines, and four adapter packages (core, web, CLI, MCP). The architecture is sound. The code compiles. The docs site exists.
The Problem: Grain is still a specification talking to itself. No AI model outputs G-Lang. No production frontend renders it. The parser is regex-based. The web adapter generates custom elements no browser understands natively. There is no streaming parser, no real SSE/WebSocket integration, no production-grade rendering, and critically — no adoption flywheel.
This PRD: Defines the roadmap from “nice spec” to “the thing AI models actually output.” It covers what to build, what to fix, what to add, and most importantly — the adoption mechanics that make standards win.
